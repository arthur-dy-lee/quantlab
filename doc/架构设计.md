## 1. 设计目标(架构层面)

| 编号 | 目标 | 落地手段 |
|------|------|----------|
| A-1 | 多市场统一,上层与数据源解耦 | 适配器模式 + 统一 OHLCV schema |
| A-2 | 本地优先,可离线复现 | `DataManager` 编排 + Parquet/SQLite 缓存 |
| A-3 | 易扩展(新市场/策略/指标/规则) | 抽象基类 + 注册表,新增不改上层 |
| A-4 | 零运维,纯本地 | 嵌入式存储,无服务器进程 |
| A-5 | 缺库不崩 | 数据源适配器懒加载 |

## 2. 分层架构

```
┌────────────────────────────────────────────────────────────────┐
│  表现层 (Presentation)                                            │
│  CLI (cli.py / typer)   Dashboard (dashboard/app.py / streamlit) │
│                         Jupyter (notebooks/)                      │
└───────────────────────────────┬──────────────────────────────────┘
                                 │ 只调用 DataManager + 功能层
┌────────────────────────────────────────────────────────────────┐
│  功能层 (Domain)                                                  │
│  indicators/   backtest/   screener/   signals/   ml/             │
│  纯计算,输入标准 OHLCV / 指标,不关心数据来源                       │
└───────────────────────────────┬──────────────────────────────────┘
                                 │ 取数只经此门面
┌────────────────────────────────────────────────────────────────┐
│  编排层 (Orchestration)        ★ DataManager(门面)                │
│  history() 本地优先取数  quote() 在线行情  download()  catalog()   │
└──────────────┬───────────────────────────────┬───────────────────┘
               │                                │
┌──────────────┴───────────┐      ┌─────────────┴────────────────────┐
│  存储层 (Storage)          │      │  数据源层 (DataSource)            │
│  BarRepository            │      │  registry.get_source(market,cfg) │
│  - Parquet 一票一文件       │      │  DataSource (ABC)                │
│  - SQLite 元数据目录        │      │   ├ AkshareSource  (CN / HK)     │
│                           │      │   ├ YFinanceSource (US / HK)     │
│                           │      │   └ CcxtSource     (CRYPTO)      │
└───────────────────────────┘      └───────────────────────────────────┘
┌────────────────────────────────────────────────────────────────┐
│  基础层 (Foundation)                                              │
│  symbols.py(统一代码)        config.py(配置 / config.yaml)        │
└────────────────────────────────────────────────────────────────┘
```

**依赖方向**:自上而下单向依赖,且上层只依赖**抽象**(`DataSource` 接口 + 标准 OHLCV DataFrame),不依赖具体数据源 —— 依赖倒置原则。

## 3. 分层职责

| 层 | 模块 | 职责 | 不负责 |
|----|------|------|--------|
| 表现 | `cli.py` / `dashboard/app.py` | 交互、参数解析、结果展示 | 业务计算、取数细节 |
| 功能 | `indicators/` `backtest/` `screener/` `signals/` `ml/` | 纯计算逻辑 | 数据从哪来、怎么存 |
| 编排 | `data_manager.py` | 本地优先取数、缓存补齐、在线/离线切换 | 具体市场协议、指标算法 |
| 存储 | `storage/repository.py` | 读写 Parquet、维护 SQLite 目录、合并去重 | 联网、计算 |
| 数据源 | `datasources/` | 各市场协议 → 归一成标准 OHLCV | 缓存、业务 |
| 基础 | `symbols.py` `config.py` | 代码格式、配置加载 | —— |

## 4. 设计模式

| 模式 | 位置 | 解决什么 |
|------|------|----------|
| **适配器 (Adapter)** | `datasources/*` | 各源格式各异 → `_normalize()` 归一成统一 OHLCV DataFrame |
| **仓库 (Repository)** | `BarRepository` | 把"读/写/合并/目录"封装,上层不碰文件细节 |
| **策略 (Strategy)** | `backtest/Strategy` | 策略只产仓位,引擎负责算收益,二者正交 |
| **注册表/工厂 (Registry)** | `registry.get_source` | 按 `市场→源` 配置路由,带实例缓存 |
| **门面 (Facade)** | `DataManager` | 上层唯一取数入口,隐藏缓存+数据源协作 |
| **规则对象 (Rule)** | `screener.RULES` / `signals.DEFAULT_RULES` | 把"一个判断"做成可组合的对象 |

## 5. 核心数据流

### 5.1 `history()` —— 本地优先取历史(回测/研究)

```
调用 dm.history("US:AAPL", start="2018-01-01")
  │
  ├─ parse_symbol → Symbol(market="US", code="AAPL")
  ├─ repo.load(sym, freq) ───────────────→ 本地 Parquet(可能为空/不全)
  ├─ _needs_fetch(local, start, end)?
  │     · 本地为空            → 需补
  │     · 本地起点晚于 start   → 需补
  │     · 本地终点过期(>3天)  → 需补
  │
  ├─ 需补 且 not offline_only:
  │     get_source("US", cfg).history()  ── 联网拉取 ──→ fetched
  │     repo.save() ── 合并 + 按日期去重 + 落 Parquet + 更新 SQLite 目录
  │
  ├─ 需补 但 offline_only:  跳过联网,用现有本地(P-3 可复现)
  │
  └─ _slice(local, start, end) ──────────→ 返回请求区间 DataFrame
```

### 5.2 `quote()` —— 在线实时行情(实盘决策)

```
dm.quote("CN:600519")
  ├─ offline_only?  是 → 退化为最近一根本地收盘价(带 note 标注)
  └─ 否 → get_source(market, cfg).quote() → 在线延迟快照 {price, time, ...}
```

> 设计意图:**历史走缓存、实时永远在线**,对应"本地回归 + 在线决策"两个诉求。

## 6. 数据模型(统一契约)

整个系统的"普通话"是一种标准 OHLCV DataFrame —— 所有数据源 `_normalize()` 后都长这样,所有功能层都只认它:

```
index : DatetimeIndex  (name="date", 升序, 去重, 无时区)
columns: open, high, low, close, volume   (float64)
```

代码统一格式(`symbols.py` 的 `Symbol`):

```
市场:代码     CN:600519  US:AAPL  HK:00700  CRYPTO:BTC/USDT
            └ 上层全程用这个,数据源层负责翻译成各自的原生代码
```

## 7. 存储设计

### 7.1 双层存储

| 存什么 | 用什么 | 路径 | 角色 |
|--------|--------|------|------|
| 行情 OHLCV | **Parquet**(列式压缩) | `data/bars/<市场>/<代码>_<周期>.parquet` | 主数据,一票一文件(已分区) |
| 元数据目录 | **SQLite** | `data/quantlab.db` 表 `bars_meta` | 覆盖区间/行数/来源/更新时间,决定是否联网 |

### 7.2 为什么这样选

- **访问模式匹配**:回测 = 读单只票全历史 → 一文件一读直接进 DataFrame,零查询开销,比关系型 DB 更快。
- **Parquet 是数据库级格式**:列式、带类型、压缩、可按列/按行组读,不是裸 CSV。
- **零运维**:Parquet + SQLite 均嵌入式,无进程要起,契合 A-4。
- **量级够用**:实测 ~52.6 字节/行,全市场 30 年日线约 6 GB,Parquet 完全胜任(详见需求文档 §4)。

### 7.3 何时升级

| 触发条件 | 升级方案 |
|----------|----------|
| 横截面查询(某天全市场筛选) | 一票一文件要开几千文件 → 引入 **DuckDB** 直接 SQL 查 Parquet(原地查,零迁移) |
| 分钟/tick 大数据(几百 GB) | 分区 Parquet 数据集 + DuckDB |

> 升级是**叠加**而非替换:Parquet 继续当存储底座,DuckDB 作为查询层覆盖其上。

## 8. 扩展点(怎么加东西)

| 想加 | 改哪里 | 步骤 |
|------|--------|------|
| 新数据源/市场 | `datasources/` + `registry._build` | 继承 `DataSource` 实现 `history()`/`quote()`,在 registry 注册,config 里映射 |
| 新策略 | `backtest/strategies.py` | 继承 `Strategy` 实现 `generate_positions(df)→仓位序列` |
| 新指标 | `indicators/technical.py` | 加函数,(可选)挂进 `add_indicators` |
| 新选股规则 | `screener.RULES` | 加一个 `TechnicalRule(name, fn)` |
| 新盯盘信号 | `signals.DEFAULT_RULES` | 加一个 `SignalRule(name, kind, fn)` |
| 信号推送通道 | `signals.notify()` | 在钩子里接钉钉/Server酱/邮件 |

**关键**:以上每一项都**不需要改动其它层** —— 这就是分层 + 抽象的价值。

## 10. 技术选型理由

| 选型 | 理由 | 备选 |
|------|------|------|
| Python | 数据/量化生态最全(pandas/sklearn) | —— |
| pandas | OHLCV 天然契合 DataFrame | polars(更快,未来可选) |
| Parquet | 列式压缩、读取快、嵌入式 | CSV(慢/无类型)、HDF5 |
| SQLite | 嵌入式、零运维,做元数据目录够用 | —— |
| akshare/yfinance/ccxt | 免费、无需付费 key、覆盖全 | tushare(部分需积分) |
| typer | 简洁声明式 CLI | argparse / click |
| streamlit | 几十行出本地仪表盘 | dash / gradio |

## 11. 已知约束与演进路径

**当前架构的边界(详见需求文档 §9 Backlog):**
- 回测为**单标的**,无组合/资金管理/滑点模型 → OPT-9
- 增量补数"全有或全无",未做缺口级补齐 → OPT-2
- A股前复权(qfq)缓存有"缝合怪"风险 → OPT-1(改 hfq 或存原始价+因子)
- 取数串行、无内存缓存、无重试 → OPT-3/5/6

**演进方向:**
1. 存储:Parquet → + DuckDB 查询层(横截面/分钟数据)
2. 回测:单标的 → 组合回测 + 资金管理
3. 数据:加 baostock(A股兜底)、基本面财务接口
4. 工程:并发取数、内存缓存、重试退避、结构化日志

> 这些都是在现有分层上**叠加**,不需要重构骨架。
