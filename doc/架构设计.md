| 项 | 内容 |

## 4. 设计模式

| 模式 | 位置 | 解决什么 |
|------|------|----------|
| **适配器 (Adapter)** | `datasources/*`、`notify/*` | 各源/各通道格式各异 → 归一成统一 OHLCV / 统一 Notifier |
| **仓库 (Repository)** | `BarRepository` | 封装"读/写/合并/目录"，上层不碰文件细节 |
| **策略 (Strategy)** | `backtest/Strategy`、`stats/PositionSizer` | 策略只产仓位/仓位算法可替换，与引擎正交 |
| **注册表/工厂 (Registry)** | `registry.get_source`、`notify` 通道表 | 按配置路由(市场→源、通道名→Notifier)，带实例缓存 |
| **门面 (Facade)** | `DataManager` | 上层唯一取数入口，隐藏缓存+数据源协作 |
| **规则对象 (Rule/Predicate)** | `screener.RULES`、`signals.DEFAULT_RULES`、`stats.conditions` | 把"一个判断/条件"做成可组合的对象 |
| **门面+薄封装 (Wrapper)** | `indicators`、`backtest.report` | 第三方库(pandas-ta/quantstats)藏在自有接口后，可换(A-6/NFR-6) |

## 5. 核心数据流

### 5.1 `history()` —— 本地优先取历史(回测/研究/统计)

```
dm.history("US:AAPL", start="2018-01-01", adjust="qfq")
  ├─ parse_symbol → Symbol(market="US", code="AAPL", instrument_type=stock)
  ├─ repo.load(sym, freq) ───────────────→ 本地 Parquet(原始价, 可能为空/不全)
  ├─ _needs_fetch(local, start, end)?  (本地空 / 起点晚于 start / 终点落后于该市场最后交易日)
  │     └─ "过期"用 calendar.last_trading_day(market) 判定，非固定天数(周末/节假日/各市场休市不同)
  │     └─ 需补 且 not offline_only:
  │           get_source(market,cfg).history()  ── 联网拉取(原始价+复权事件)
  │           repo.save() ── 合并 + 按日期去重 + 落 Parquet + 更新 SQLite 目录 + 存因子
  │     └─ 需补 但 offline_only:  跳过联网，用现有本地(P-3)
  ├─ adjust.apply(raw_df, factors, mode="qfq")  ── 读取时现算复权(§8)
  └─ _slice(start, end) ──────────────────→ 返回请求区间 DataFrame
```

### 5.2 `quote()` —— 在线(延迟)行情

```
dm.quote("CN:600519")
  ├─ offline_only? 是 → 退化为最近一根本地收盘价(带 note 标注)
  └─ 否 → get_source(market,cfg).quote() → 在线延迟快照 {price, time, ...}
```

> 设计意图：**历史走缓存、实时永远在线** —— 对应"本地回归 + 在线决策"两个诉求。

### 5.3 条件概率统计(FR-9.1) —— ★核心

```
stats.probability(symbol="CN:000300", condition, forward=1)
  ├─ df = dm.history(symbol)              # 本地优先取历史
  ├─ df = indicators.add(df)              # 需要 RSI 等时
  ├─ mask = condition(df)                 # 条件谓词: 跌幅>1.5% / 连跌2天 / RSI<30 …(可组合)
  ├─ fwd = forward_return(df, n=forward)  # 防未来函数: 用 mask 当日之后的收益
  ├─ base = 无条件次日上涨率(全样本)        # 基准率, 用于算 edge
  └─ 汇总 fwd[mask] → { 样本数 N, 上涨概率 p, edge=p−base, p 的置信区间(Wilson), 平均/中位涨跌幅, 盈亏比 b, MAE }
                       └─ N 小 → 置信区间自然变宽;N < 阈值 → 标记"不可靠"(P-7)
```

### 5.4 跨市场领先预警(FR-9.2) —— 美股 → A股

```
stats.crossmarket(lead="US:^IXIC", target="CN:000300", forward=1)
  ├─ lead_df  = dm.history(lead)          # 美股(昨夜已收盘)
  ├─ tgt_df   = dm.history(target)        # A股
  ├─ calendar.align(lead, target, lag=1)  # 以 A股交易日为轴, 取其前一个已完成的美股 session
  │     (美股 day-T 收盘发生在 A股 day-(T+1) 开盘前 → 信息可用，无未来函数)
  ├─ mask = lead_return < -1.5%           # 条件作用在 lead 上
  └─ 汇总 A股次日收益[mask] → { N, 跟跌概率, 平均跌幅, ... } → 盘前预警卡片
```

### 5.5 信号 → 通知(FR-6 / FR-9.4)

**职责边界**：`signals` 是**实时触发层**(判定"当下某条件成立")，`stats` 是**概率标注层**(给该条件的历史概率/edge/仓位/风险)。二者组合产出卡片：

```
signals.scan(watchlist)                      # 触发层: 逐标的判定当下条件是否成立
  └─ 命中条件 c → stats.annotate(symbol, c)   # 标注层: 该条件的历史概率/基准率/edge/仓位/风险
       └─ 产出 SignalCard{ symbol, 触发条件 c, 上涨概率+edge, 样本数, 建议仓位, 风险, 时间 }
            └─ notify.dispatch(card, channels=["feishu"])
                 └─ registry[channel].send(card)   # FeishuNotifier: 卡片 + 加签 + POST webhook
```

### 5.6 盯盘扫描(`watch`) —— cron 触发的一次性扫描

零运维(P-5)下不常驻进程:`quantlab watch` **跑完一轮扫描即退出**,由系统 cron/launchd 定时触发(如盘中每 15 分钟、盘前一次)。

```
cron/launchd → quantlab watch   (一次性, 跑完退出)
  ├─ 对 watchlist 逐标的:取数(延迟行情/最新日线) → signals/stats 检查
  ├─ 命中 → 组装 SignalCard
  ├─ notify_log 去重:同标的同信号在 throttle 窗口内已推过 → 跳过
  └─ notify.dispatch(card) → 飞书等通道,并写 notify_log
```

> 这解决了"零运维(无常驻进程)"与"异动周期性通知"的张力:进程不常驻,定时性交给操作系统。

## 6. 数据模型(统一契约)

整个系统的"普通话"是标准 OHLCV DataFrame —— 数据源 `_normalize()` 后都长这样，功能层只认它：

```
index   : DatetimeIndex (name="date", 升序, 去重, 无时区)
columns : open, high, low, close, volume   (float64)         # 存原始价
```

`Symbol`(基础层)：

```
Symbol(market, code, instrument_type)
  market         : CN | US | HK | CRYPTO
  code           : 原生代码 (600519 / AAPL / 00700 / BTC/USDT)
  instrument_type: stock | etf | index | crypto
                   # A股按代码段判定(沪51/56/58·深15→etf);US 代码分不出 AAPL/QQQ,
                   # 靠数据源标志位或维护清单;crypto 按前缀。判定结果落库
  字符串形态      : "CN:600519"  上层全程用它，数据源层翻译成原生代码
```

完整对象契约见需求文档 §4.4。要点：行情存**原始价**；复权因子单独存；ETF 专有字段(nav/折溢价/theme/constituents)做可空子表，仅 ETF 填。

> `_normalize()` 同时做**数据质量校验**：处理 A股停牌缺口、丢弃/标记 0 成交量或异常坏值，避免污染下游指标与概率。

## 7. 存储设计

### 7.1 双层存储

| 存什么 | 用什么 | 路径 | 角色 |
|--------|--------|------|------|
| 行情 OHLCV(**原始价**) | **Parquet** | `data/bars/<市场>/<代码>_<周期>.parquet` | 主数据，一票一文件 |
| 复权因子(分红/拆分) | Parquet/独立列 | `data/adjust/<市场>/<代码>.parquet` | 现算 qfq/hfq 的依据 |
| 元数据目录 | **SQLite** | `data/quantlab.db` 表 `bars_meta` | 覆盖区间/行数/来源/更新时间，决定是否联网 |
| ETF 扩展元数据 | SQLite/可空表 | `data/quantlab.db` 表 `etf_meta` | nav/折溢价/theme/constituents |

### 7.2 为什么这样选

- **访问模式匹配**：回测/统计 = 读单只票全历史 → 一文件一读直接进 DataFrame，零查询开销。
- **Parquet 是数据库级格式**：列式、带类型、压缩，可按列/行组读，远胜裸 CSV。
- **零运维**：Parquet + SQLite 均嵌入式，无进程要起(A-7)。
- **量级够用**：全市场 30 年日线约 6 GB，Parquet 完全胜任(需求文档 §4.2)。
- **多周期单一真相**：日线为唯一落盘真相，周/月线**本地从日线重采样**(不重复存/拉、口径一致);加密的小时/分钟线才直接向源拉取并单独落盘。

### 7.3 何时升级

| 触发条件 | 升级方案 |
|----------|----------|
| 横截面查询(某天全市场筛选) | 引入 **DuckDB** 直接 SQL 查 Parquet(原地查，零迁移) — OPT-10 |
| 分钟/tick 大数据(几百 GB) | 分区 Parquet 数据集 + DuckDB |

> 升级是**叠加**而非替换：Parquet 继续当存储底座，DuckDB 作查询层覆盖其上。

## 8. 复权设计(关键正确性决策)

**决策**：缓存层只存**原始价 + 复权因子**，不固化任何复权方式；读取时按 `adjust=raw|qfq|hfq` 由 `adjust.py` **现算**。

- qfq/hfq 都是原始价的纯函数 → 缓存永远是稳定的原始事实，**不会随新数据被改写**，根除旧"缝合怪"风险。
- 离线模式下复权同样可复现(P-3)。
- ETF 的"分配(distribution)"当作现金分红同等喂进同一复权管线，**不另建逻辑**。
- **因子来源**：由 DataSource 适配器产出 `(原始价 + 复权事件/因子)` —— yfinance 直接给 dividends/splits;akshare 需从分红送配接口拼装;`instrument_type ∈ {crypto, index}` 时复权为**恒等**(无除权除息)。
- **实时决策口径**："现在"的盯盘信号/指标统一用 **qfq**(qfq 锚定最新价 → 最新 K 线 ≈ 原始价，与在线 `quote()` 报价口径一致);hfq 仅用于长周期回测/绝对价位研究。

数据流见 §5.1(`adjust.apply` 步骤)。详细动机见需求文档 §6。

## 9. 统计概率引擎设计(`stats/`) —— ★产品核心

这是 QuantLab 区别于普通回测工具的地方：不预测、不自动交易，用**历史条件频率**回答"此情此景，接下来涨/跌的概率多大"，并给出仓位与风险供人判断。

### 9.1 模块拆分

| 文件 | 职责 |
|------|------|
| `conditions.py` | 条件谓词：`drop_gt(x)` / `n_down_days(n)` / `rsi_lt(t)` / `vol_spike()` …，返回布尔 mask，**可用 & \| 组合** |
| `probability.py` | 给定 mask + 前向窗口 N → 统计 {样本数, 上涨概率, 平均/中位涨跌幅, 盈亏比, MAE} |
| `crossmarket.py` | 跨市场对齐(日历 + 滞后) → 用 lead 市场条件统计 target 市场反应 |
| `sizing.py` | `PositionSizer` 策略：由胜率 p、盈亏比 b 给建议仓位 |

### 9.2 防未来函数(对齐 FR-4.2)

- 条件在 `t` 成立 → 只用 `t` **之后**的收益评估；前向收益 `fwd = close.shift(-N)/close - 1`。
- 跨市场：以 A股交易日为轴，条件取**前一个已收盘**的美股 session(信息在 A股开盘前已知)。
- **共享原语**：前向收益/无未来函数逻辑是全系统最易错处，抽到中性模块 `core.py`(`forward_return()` 等)，由 `stats` 与 `backtest` **共用、共测**，杜绝两处实现不一致。

### 9.3 仓位与风险(FR-9.3) —— `PositionSizer` 可替换

提供两个内置实现，`config.yaml` 选用(回避"先固定档位还是凯利"的悬而未决)：

- `FixedTierSizer`(默认)：按上涨概率映射到固定档位(如 p≥0.6→30%、p≥0.7→50%)，直观可控。
- `KellyConservativeSizer`：`f = (b·p − (1−p)) / b`，再乘保守系数(如 0.25~0.5)并设上限。

输出一律附：上涨概率、样本数、盈亏比、MAE(下行风险)、建议仓位上限 —— 仅参考，不自动执行(P-6)。

### 9.4 诚实评估(P-7)

- **对比基准率(关键)**：总把条件概率与**无条件基准率**(该标的/指数全样本的次日上涨率)并列，报 **edge = 条件概率 − 基准率**。脱离基准率的"上涨概率 60%"会误导(若基准本就 55%，则几乎无 edge)。
- **置信区间**：给概率的 Wilson 置信区间；样本越少区间越宽，直观体现不确定性。
- 样本数 < 阈值(默认 30)标记"不可靠"；展示区间与样本量；不做样本内调参后又用同段吹概率。

## 10. 通知设计(`notify/`)

### 10.1 结构

```
notify/
  base.py    : Notifier(ABC).send(card)  +  SignalCard 数据类  +  注册表 dispatch(card, channels)
  feishu.py  : FeishuNotifier  ★首选  (webhook + 可选加签 + 交互式消息卡片)
  console.py : ConsoleNotifier (默认, 本地打印)
  (telegram.py / email.py : 留位, M3)
```

- 统一钩子 `notify.dispatch(card, channels=[...])`；新增通道只实现一个 `Notifier`，注册即用(A-5)。
- `SignalCard` 字段：标的、触发条件、上涨概率(+基准率/edge/置信区间)、样本数、建议仓位、风险提示(MAE)、时间。

### 10.2 飞书要点(实现前以官方文档为准)

- 自定义机器人是**群机器人**：需先建群(仅自己也行)、加机器人拿 webhook；个人免费账号即可。
- 加签：`sign = base64(HMAC-SHA256(key=f"{ts}\n{secret}", msg=""))`，请求带 `timestamp`+`sign`。
- 限流：约 100 条/分钟(待核实) → 通知做**节流/去重**(同标的同信号 N 分钟内不重复推)。
- 去重状态**跨进程持久化**：推送记录写 SQLite 表 `notify_log`(symbol+signal+时间);`watch` 经 cron 每次是新进程,靠它判断 throttle 窗口,避免重复推。
- secret/webhook 放 `config.yaml` 或环境变量，不入库、不进 git。

| 库 | 接入方式 | 藏在哪个接口后 |

## 18. 测试策略

核心逻辑全部用**合成数据离线**测试(NFR-5),不依赖联网:

| 测什么 | 怎么测 |
|--------|--------|
| 指标 | 构造已知序列，断言 MA/RSI/MACD 等数值与手算一致 |
| 回测引擎 | 喂确定性价格 + 固定仓位，断言收益/净值/回撤可算准；**重点验证无未来函数**(信号次日生效) |
| **统计概率(stats)** | 构造已知"涨跌图案"，断言条件 mask 命中位置正确、前向收益用的是条件之后的数据(**无未来函数**)、概率/盈亏比/MAE 数值正确 |
| 跨市场对齐 | 构造两市场错峰序列，断言"美股 day-T → A股 day-(T+1)"的对齐与滞后正确，无前视 |
| DataManager 本地优先 | 用 **FakeDataSource**(返回合成数据、可记录调用次数)，断言:本地命中不联网、缺口才联网、`offline_only` 绝不联网(OPT-7) |
| 复权 | 给定原始价+已知分红/拆分，断言 qfq/hfq 现算结果正确，且增量更新后历史不被改写(无"缝合") |
| 通知 | Mock webhook，断言加签正确、`notify_log` 去重在 throttle 窗口内生效 |

> 最关键的两条:**无未来函数**(回测 + stats + 跨市场)、**离线可复现**(FakeDataSource + offline_only)。
