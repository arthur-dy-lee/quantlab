## 1. 项目背景与目标

构建一款**完全在本地运行**的股票分析软件,核心目的是 **发现投资机会,并在合适时机通知我手动操作**。

核心诉求:

- **机会发现 + 通知**:用历史数据做**统计/条件概率**判断(如"跌得多了次日反弹的概率"),命中条件就推送通知,带上概率、建议仓位与风险,**由我自己拍板下单**;
- **主战场 A 股**;**美股作为领先指标**——美股大跌往往领先 A 股,用它提前预警 A 股可能的波动,提前发现潜在标的;
- 能拿到**历年免费**的股票数据,**本地存一份**用于回测与回归验证;实盘决策时读取**在线免费**(延迟)行情;
- 覆盖多个市场,在同一套工具里完成研究 → 回测 → 概率统计 → 选股 → 盯盘通知。

> 定位:**个人研究与决策辅助 + 信号通知工具**。
> 明确不做:**自动下单 / 券商对接**、**高频 / 短线 / 超短线**。信号只提示,执行由人决定。

---

## 2. 核心原则(贯穿全局的设计约束)

| 编号 | 原则 | 说明 |
|------|------|------|
| P-1 | 本地优先 | 数据本地缓存一份,回测/研究默认走本地,不依赖联网 |
| P-2 | 全免费数据源 | 不使用任何需付费的 API key |
| P-3 | 可复现 | 回测结果可复现(离线模式锁定本地数据;复权按需现算不固化) |
| P-4 | 多市场统一 | A股/美股/港股/加密货币走同一套数据接口,上层与数据源解耦 |
| P-5 | 零运维 | 纯本地,无需启动服务器/数据库进程 |
| P-6 | 人本决策 | 信号只**提示**不自动执行;每个信号都给出概率、样本量、收益与风险、建议仓位,供人判断 |
| P-7 | 诚实可解释 | 概率基于历史频率,标注样本量与局限,不夸大;样本太少明确提示不可靠 |

## 3. 功能需求

> 每条 FR 标注所属里程碑(M1 薄主干 / M2 研究闭环 / M3 深化),里程碑定义见 §8。

### 3.1 数据获取(FR-1) — M1
- FR-1.1 支持按统一代码格式取数:`CN:600519` / `US:AAPL` / `HK:00700` / `CRYPTO:BTC/USDT`
- FR-1.2 历史 K 线:日线为主,支持周线/月线;加密货币额外支持小时/分钟线
- FR-1.3 实时(延迟)行情快照:用于实盘决策
- FR-1.4 批量下载并缓存到本地
- FR-1.5 **复权**:股票类标的存**原始价 + 复权因子**(分红/拆分事件),前复权(qfq)/后复权(hfq)在读取时按需现算,缓存层**不固化**某种复权方式 —— 从源头避免缓存"缝合怪"(详见 §6 复权设计)

### 3.2 数据存储(FR-2) — M1
- FR-2.1 历史行情存为 **Parquet**(列式压缩),按 `市场/代码/周期` 分文件
- FR-2.2 **SQLite** 仅存元数据目录(覆盖区间、行数、来源、更新时间)
- FR-2.3 本地缺失区间时自动联网补齐并合并去重落盘
  - M1 可先做"全有或全无"补齐;缺口级精确补齐见 OPT-2(M1 之后)
- FR-2.4 `offline_only` 开关:纯离线模式,只读本地、绝不联网(保证 P-3 可复现)

### 3.3 技术指标(FR-3) — M1
- 提供 MA / EMA / RSI / MACD / 布林带 / ATR,可一键给 OHLCV 追加全套指标列

### 3.4 回测(FR-4) — M1
- FR-4.1 策略模式:策略只产"目标仓位",引擎负责算收益/净值/绩效
- FR-4.2 防未来函数(信号次日生效)
- FR-4.3 计入交易成本(可配置基点)
- FR-4.4 输出:总收益、年化(CAGR)、波动、夏普、最大回撤、交易次数、买入持有对照
- FR-4.5 内置示例策略:双均线、RSI 均值回归
- FR-4.6 当前为**单标的**回测;组合(多标的)回测 + 资金管理见 OPT-9(M3)

### 3.5 选股筛选(FR-5) — M2
- FR-5.1 技术面规则筛选(金叉、超卖、站上均线、MACD 红柱…),规则可自由组合
- FR-5.2 基本面筛选(PE/PB/ROE/财报)—— **规划中(M3),接口已留位**
- FR-5.3 标的池支持按主题维护的 watchlist(见 §4.3 ETF/主题标的池)

### 3.6 盯盘信号与通知(FR-6) — M2
- FR-6.1 对一篮子标的(watchlist)检查买卖信号(RSI 超卖/超买、跌破均线、MACD 金叉、FR-9 概率信号…)
- FR-6.2 信号推送:**飞书自定义机器人(消息卡片)为首选通道**;钉钉 / Telegram / Server酱 / 邮件(SMTP)为备选;默认本地打印
  - 飞书:需先建群并把机器人加进去拿到 webhook(个人账号即可,免费);支持加签;约 100 条/分钟限流(实现时以官方文档为准,详见 §12 调研)
  - 推送接口做成统一钩子 `notify(channel, card)`,新增通道不动上层(见架构扩展点)

### 3.7 机器学习预测(FR-7) — M3
- FR-7.1 用历史指标做特征,预测下一根 K 线涨跌(示例骨架,扩展位)
- FR-7.2 时序切分训练/测试,报告相对多数类基准的超额(诚实评估,不只看准确率)

### 3.8 使用界面(FR-8)
- FR-8.1 **CLI**(M1):download / quote / backtest / catalog;(M2)screen / watch / dashboard;(M3)predict
- FR-8.2 **本地 Web 仪表盘 / 图表**(Streamlit + plotly,M2):**交互式**(缩放/悬停)展示 —— K 线蜡烛图 + 指标叠加、回测净值与回撤曲线、FR-9 概率统计分布;一键回测。
  - M1 阶段 CLI 只输出**数字/表格,不出图**;图表能力统一放在 M2(按你的选择:一次性上交互看板)
- FR-8.3 **Jupyter**(M3):研究/调参/自定义策略(提供 quickstart notebook)

### 3.9 统计概率信号引擎(FR-9) — ★核心 — M1(单市场) / M2(跨市场+通知)

本项目的核心价值:不预测、不自动交易,而是用**历史条件频率**告诉我"此情此景,接下来涨/跌的概率有多大",命中就通知,带上概率与风险供我判断。

- **FR-9.1 条件频率统计(M1)**:给定一个历史条件,统计其后 N 日(默认次日)的表现。
  - 条件示例:单日跌幅 > X%、连续下跌 N 天、RSI < 阈值、放量下跌、跌破均线…(条件可组合)
  - 输出:**样本数、上涨概率、平均涨跌幅、中位涨跌幅、盈亏比、最大不利波动(MAE)**
  - 落地示例(你的用例):
    - "沪深300 单日跌幅 > 1.5% → 次日上涨概率 = ? 平均涨幅 = ?"
    - "某标的连续下跌 2 天 → 第 3 天上涨概率 = ?"
- **FR-9.2 跨市场领先预警(M2)**:用 A 市场的事件作条件,统计 B 市场的反应。
  - 落地示例(你的用例):"美股(纳指/标普)昨夜跌幅 > 1.5% → 次日 A 股(沪深300)跟跌的概率 / 平均跌幅 = ?" → 盘前生成预警
- **FR-9.3 建议仓位与风险(M2)**:基于胜率 p 与盈亏比 b 给出**参考仓位**(凯利分数的保守折扣版,或映射到固定档位如 10/30/50%,上限可配),并同时列出收益概率与下行风险。**仅供参考,不自动执行(P-6)**。
- **FR-9.4 通知(M2)**:命中信号经 §3.6 推送通道(飞书优先)发出,卡片含:标的、触发条件、上涨概率、样本数、建议仓位、风险提示。
- **FR-9.5 诚实评估(P-7)**:标注样本量;样本过少(如 < 30)标记"不可靠";避免幸存者偏差与未来函数;概率是历史频率非预测保证。

> 技术上这是 pandas 的条件分组频率统计,几十行即可实现,无需依赖外部脆弱 app,且完全可复现(见 §12 调研结论)。

### 4.3 ETF / 主题标的池(建模决策)

**决策:ETF 不单列资产类,以"统一 `Symbol` + `instrument_type` 类型标记"承载。** ETF 在 K线/技术指标/概率信号/回测/watchlist 上与个股 100% 同构,差异只在少数基本面与特有字段,用"类型标记 + 可空字段 + 规则跳过"处理即可,独立类只会带来重复代码。主流平台(qlib/akshare/yfinance)亦如此。

- **类型标记**:每个 `Symbol` 带 `instrument_type ∈ {stock, etf, index, crypto}`,入库时按交易所+代码段自动判定(A股 ETF:沪 `51/56/58xxxx`、深 `15xxxx`;个股:沪 `6xxxxx`、深 `000/002/30xxxx`),判定结果落库。
- **复用管线**:OHLCV、技术指标、FR-9 概率信号、回测对 ETF 与个股完全一致;ETF 的"分配(distribution)"按现金分红同等走 §6 复权管线,**不另建逻辑**。
- **可空元数据**:ETF 专有字段(`nav/iopv`、`折溢价`、`tracking_index`、`theme[]`、`constituents[]`)做成可空/子表,仅 ETF 填,个股留空。
- **选股筛选**:基本面规则(PE/PB/ROE)标 `stock-only`,遇 ETF **跳过且不判负**(勿因字段空而误淘汰),UI 提示"已跳过 N 条基本面规则"。技术面/动量/概率规则对全类型生效。
- **主题 watchlist**:ETF 与个股同列一个池,按 `theme[]` 打标(科技 / 机器人 / AI / 智能制造 等);ETF 直接打标(如 `CN:159819→机器人`),在 `config.yaml` 配置,作为选股(FR-5)与盯盘(FR-6)的标的池。
- (可选,非 MVP)对 ETF 增加"折溢价偏离过大"提示信号,利用 NAV/IOPV 字段。

### 4.4 数据模型概览(数据契约)

> 详细 schema 待架构文档重做时展开;此处先把跨层的数据契约定下来。

| 对象 | 形态 | 关键字段 |
|------|------|----------|
| 统一 OHLCV | pandas DataFrame | `DatetimeIndex`(date,升序去重) + open/high/low/close/volume(float) |
| Symbol | 值对象 | `market`(CN/US/HK/CRYPTO) + `code` + `instrument_type`(stock/etf/index/crypto) |
| 行情存储 | Parquet,一票一文件 | `data/bars/<市场>/<代码>_<周期>.parquet`,存**原始价** |
| 复权因子 | 序列/事件表 | 由分红/拆分推算的 factor,与原始价一起现算 qfq/hfq(§6) |
| 元数据目录 | SQLite 表 `bars_meta` | symbol/freq/覆盖区间/行数/来源/更新时间 → 决定是否联网补齐 |
| ETF 扩展(可空) | 子表/可空列 | nav/iopv、折溢价、tracking_index、theme[]、constituents[] |
| watchlist/主题 | `config.yaml` | 主题 → 标的列表(个股 + ETF 混合) |
| 概率统计结果 | DataFrame/字典(可不持久化) | 条件、样本数、上涨概率、平均/中位涨跌幅、盈亏比、MAE |

**存储选型**:行情 **Parquet** + 元数据 **SQLite**,均嵌入式、无服务进程(P-5);无需 MySQL/Postgres。横截面/分钟大数据时**叠加** DuckDB(OPT-10),不替换 Parquet。

## 6. 复权设计(关键正确性决策)

**问题**:前复权(qfq)随"最新价"为锚,新数据到来时整条历史会被重新缩放;若把 qfq 结果直接落盘缓存,增量更新会拼出新旧两套缩放基准混合的"缝合怪"序列,破坏可复现性(P-3)。

**决策**:缓存层**只存原始价 + 复权因子**(由分红/拆分事件推算),不固化任何复权方式。

- 读取时按调用方请求 `adjust=raw|qfq|hfq` **现算**复权价;
- qfq/hfq 都是原始价的纯函数,缓存永远是稳定的原始事实,不会随新数据被改写;
- 离线模式下复权同样可复现。

> 这取代了旧文档中遗留的 OPT-1(qfq 缝合)风险 —— 从架构上不再发生。

## 7. 约束与假设

- C-1 **免费 = 延迟行情**(通常约 15 分钟,加密货币近实时)。满足决策辅助与回测,**不适合高频交易**。
- C-2 **地理限制**:Binance 对部分地区有 HTTP 451 封锁,默认加密交易所用 kraken,可在 `config.yaml` 切换。
- C-3 **不构成投资建议**,纯学习研究用途。
- C-4 数据源为第三方免费接口,接口变动/限流风险需自行容错(重试退避见 OPT-6)。

## 9. 演进 Backlog(M1 之后,按优先级)

## 11. 术语

- **OHLCV**:开/高/低/收/量,标准 K 线数据
- **前复权/后复权(qfq/hfq)**:股价除权除息的调整方式;前复权锚定最新价、后复权锚定首日价。本项目缓存只存原始价+复权因子,两者均按需现算(§6)
- **复权因子**:由分红/拆分事件推算的乘数,把原始价换算为复权价
- **横截面查询**:在某一时间点跨所有标的查询(如"今天所有 RSI<30 的股票")
- **条件概率/胜率**:历史上满足某条件后,下一段时间上涨的样本占比
- **盈亏比**:平均盈利幅度 / 平均亏损幅度;与胜率共同决定仓位建议
- **MAE(最大不利波动)**:持有期内价格相对入场的最大浮亏,衡量下行风险
- **凯利分数**:由胜率与盈亏比算出的理论最优仓位比例,本项目用其保守折扣版作仓位建议
- **领先指标**:一个市场/资产的变动常先于另一个(本项目用美股领先 A 股)
- **`instrument_type`**:标的类型 `{stock, etf, index, crypto}`,区分个股/ETF/指数/加密

### 12.2 "下跌→次日上涨概率提醒" 有没有现成 app?

**没有公认的现成 turnkey 项目**专门做这件事并推送通知;这类条件频率统计通常是 DIY notebook。但你的思路有成熟理论基础——属于**短期均值回归 / "buy the dip"**(如 Larry Connors 的 RSI(2) 策略、学界 short-term reversal 研究)。

**好消息**:这正是用 pandas 条件分组几十行就能实现的统计(FR-9),自己写反而**最贴合需求、可复现、可解释**,不必依赖脆弱的第三方 app。

### 12.3 飞书通知可行性

- **可行,个人账号免费即可**。飞书"自定义机器人"是群机器人:**需先建一个群(只有自己也行)、把机器人加进去**拿 webhook。无需企业付费版。(此点是与 Telegram/Server酱"无需建群"方案的关键区别)
- **消息卡片(interactive)** 支持 markdown + 双列 fields,足够展示 标的/信号/概率/仓位/风险。
- **加签**:`sign = base64(HMAC-SHA256(key=f"{timestamp}\n{secret}", msg=""))`,请求带 `timestamp`+`sign`,时间戳偏差过大会被拒。
- **限流**:约 100 条/分钟(待官方核实)。
- **备选**:Server酱(最简单、直推微信)、邮件(最通用、实时性差)、钉钉(同需建群)、Telegram(境内需代理)。
