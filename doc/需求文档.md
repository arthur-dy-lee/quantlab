## 1. 项目背景与目标

构建一款**完全在本地运行**的股票分析软件,辅助投资决策。核心诉求:

- 能拿到**历年免费**的股票数据,**本地存一份**用于策略回测与回归验证;
- 实盘决策时能读取**在线免费**行情;
- 覆盖多个市场,在同一套工具里完成研究 → 回测 → 选股 → 盯盘。

> 定位:**个人研究与决策辅助工具**,非交易执行系统,不接券商下单。

---

## 2. 核心原则(贯穿全局的设计约束)

| 编号 | 原则 | 说明 |
|------|------|------|
| P-1 | 本地优先 | 数据本地缓存一份,回测/研究默认走本地,不依赖联网 |
| P-2 | 全免费数据源 | 不使用任何需付费的 API key |
| P-3 | 可复现 | 回测结果可复现(离线模式锁定本地数据) |
| P-4 | 多市场统一 | A股/美股/港股/加密货币走同一套数据接口,上层与数据源解耦 |
| P-5 | 零运维 | 纯本地,无需启动服务器/数据库进程 |

---

## 3. 功能需求

### 3.1 数据获取(FR-1)
- FR-1.1 支持按统一代码格式取数:`CN:600519` / `US:AAPL` / `HK:00700` / `CRYPTO:BTC/USDT`
- FR-1.2 历史 K 线:日线为主,支持周线/月线;加密货币额外支持小时/分钟线
- FR-1.3 实时(延迟)行情快照:用于实盘决策
- FR-1.4 批量下载并缓存到本地

### 3.2 数据存储(FR-2)
- FR-2.1 历史行情存为 **Parquet**(列式压缩),按 `市场/代码/周期` 分文件
- FR-2.2 **SQLite** 仅存元数据目录(覆盖区间、行数、来源、更新时间)
- FR-2.3 本地缺失区间时自动联网补齐并合并去重落盘
- FR-2.4 `offline_only` 开关:纯离线模式,只读本地、绝不联网(保证 P-3 可复现)

### 3.3 技术指标(FR-3)
- 提供 MA / EMA / RSI / MACD / 布林带 / ATR,可一键给 OHLCV 追加全套指标列

### 3.4 回测(FR-4)
- FR-4.1 策略模式:策略只产"目标仓位",引擎负责算收益/净值/绩效
- FR-4.2 防未来函数(信号次日生效)
- FR-4.3 计入交易成本(可配置基点)
- FR-4.4 输出:总收益、年化(CAGR)、波动、夏普、最大回撤、交易次数、买入持有对照
- FR-4.5 内置示例策略:双均线、RSI 均值回归

### 3.5 选股筛选(FR-5)
- FR-5.1 技术面规则筛选(金叉、超卖、站上均线、MACD 红柱…),规则可自由组合
- FR-5.2 基本面筛选(PE/PB/ROE/财报)—— **规划中,接口已留位**

### 3.6 盯盘信号(FR-6)
- FR-6.1 对一篮子标的检查买卖信号(RSI 超卖/超买、跌破均线、MACD 金叉…)
- FR-6.2 信号推送钩子(钉钉/Server酱/邮件)—— **留位,默认打印**

### 3.7 机器学习预测(FR-7)
- FR-7.1 用历史指标做特征,预测下一根 K 线涨跌(示例骨架,扩展位)
- FR-7.2 时序切分训练/测试,报告相对多数类基准的超额(诚实评估,不只看准确率)

### 3.8 使用界面(FR-8)
- FR-8.1 **CLI**:download / quote / backtest / screen / watch / predict / catalog / dashboard
- FR-8.2 **本地 Web 仪表盘**(Streamlit):看 K 线+指标、一键回测、看净值曲线
- FR-8.3 **Jupyter**:研究/调参/自定义策略(提供 quickstart notebook)

## 4. 数据需求(市场 × 数据源)

| 市场 | 数据源(库) | 历史 | 实时(免费=延迟) | 备注 |
|------|------------|------|------------------|------|
| A股 / 港股 | akshare | ✅ | ✅ 延迟 | 国内首选,无需注册 |
| A股备用 | baostock | ✅ | ❌ | 留作兜底(未接) |
| 美股 / 港股备用 | yfinance | ✅ 数十年 | ✅ 延迟约 15min | 雅虎财经 |
| 加密货币 | ccxt | ✅ | ✅ 近实时 | 接交易所公开 API |

**数据量级评估(实测 ~52.6 字节/行外推):**

| 场景 | 数据量 | 占用 | 结论 |
|------|--------|------|------|
| 几百只票日线 | ~百万行 | 几十 MB | Parquet 足够 |
| 全市场 30 年日线 | ~1.2 亿行 | ~6 GB | Parquet 足够且快 |
| 全市场分钟线 10 年 | ~百亿行 | 几百 GB | 需引入 DuckDB 查询层 |
| 主流ETF所有数据 | ~百亿行 | 几百 GB | 需引入 DuckDB 查询层 |

ETF科技相关，机器人，AI ，智能制造等

## 5. 非功能需求

- NFR-1 **运行环境**:本地,Python ≥ 3.12(开发环境为 3.12);独立 conda环境，创建 quantlab_312 环境
- NFR-2 **可扩展**:新增市场/数据源只实现一个 `DataSource` 适配器,不动上层
- NFR-3 **健壮性**:某个数据源库未安装不影响其它市场(适配器懒加载)
- NFR-4 **依赖按需**:数据源/界面依赖拆成 extras(`cn`/`us`/`crypto`/`ml`/`dashboard`)
- NFR-5 **可测试**:核心逻辑可用合成数据离线测试

## 6. 约束与假设

- C-1 **免费 = 延迟行情**(通常约 15 分钟,加密货币近实时)。满足决策辅助与回测,**不适合高频交易**。
- C-2 **地理限制**:Binance 对部分地区有 HTTP 451 封锁,默认加密交易所已改用 kraken,可在 `config.yaml` 切换。
- C-3 **不构成投资建议**,纯学习研究用途。
- C-4 数据源为第三方免费接口,接口变动/限流风险需自行容错(规划中:重试退避)。

## 9. 待优化项(Backlog,按优先级)

**正确性(优先)**
- OPT-1 🔴 A股前复权(qfq)缓存会产生"缝合怪"序列(新旧缩放基准混合)→ 改用后复权 hfq 或存原始价+复权因子
- OPT-2 🔴 增量补数为"全有或全无",应只拉缺失区间

**性能**
- OPT-3 🟡 选股/盯盘串行 → `ThreadPoolExecutor` 并发取数
- OPT-4 🟡 A股 `quote()` 为查一只票拉全市场行情表 → 缓存全表(带 TTL)/批量
- OPT-5 🟡 `history()` 与指标加内存缓存(LRU),避免重复读盘/重算

**健壮性 / 工程**
- OPT-6 🟢 数据源加重试 + 退避
- OPT-7 🟢 给 `DataManager` 本地优先逻辑补测试(用假 DataSource)
- OPT-8 🟢 引入结构化日志替代 print

**功能演进**
- OPT-9 🔵 组合回测 + 资金管理 + 滑点模型
- OPT-10 🔵 DuckDB 查询层(横截面选股 / 分钟数据)

## 10. 术语

- **OHLCV**:开/高/低/收/量,标准 K 线数据
- **前复权/后复权(qfq/hfq)**:股价除权除息的调整方式;前复权锚定最新价、后复权锚定首日价
- **横截面查询**:在某一时间点跨所有标的查询(如"今天所有 RSI<30 的股票")
