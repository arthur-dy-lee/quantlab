# 板块温度计 v1 详细设计与任务拆解

> 范围：**A股板块**（芯片/半导体、光通信、有色、黄金、白银、封装封测、消费、银行），
> 四维结构（透支/拥挤/估值/板块专属），**每板块一套配置**。美股 v2 复用合成层，仅换数据层。
> 前身：`scripts/sector_crowding.py`（价格口径雏形，本设计将其吸收升级后废弃）。
> 与大盘温度计的关系见 §1；大盘温度计设计见 `市场温度计_v1详细设计.md`。

---

## 1. 定位与设计原则

**板块温度计 ≠ 大盘温度计下沉。** 大盘 5 因子中 ERP/两融占比/证券化率都是全市场口径，
拆不到板块；周期板块（半导体/有色）的 PE 分位在景气底部反而最高（利润消失），照搬会在底部喊过热。

**定位：风险描述 + 仓位弹药参考，不是预测择时。** 这是大盘温度计已验证的结论
（前向收益对温度非单调、机械映射只控回撤），板块层面动量更强、更适用：
科创50 曾 +39% 乖离继续涨；2000 年费半冲到 MA200 上方 60%+ 后又涨数月才见顶。
高透支标记的是**脆弱性**，扳机是**拐头**。

三条硬原则（与大盘温度计一致）：

1. **无未来函数**：一切分位用 `expanding`（复用 `historical_percentile`），winsorize 用扩张窗口。
2. **缺维度按剩余权重归一化**：光通信没估值数据、白银篮子没成交占比口径，缺哪维就把权重摊给其余维度（复用 `thermometer.py` 的 `avail/wsum` 模式）。
3. **数据长度入输出**：东财概念板块多数只有 2014± 至今的历史，分位数可信度打折，报告必须标注每个板块的样本起点。

## 2. 四维结构（同时支持，按板块配权）

| 维度 | 含义 | 子指标（各自先取扩张分位 0–100，再等权/配权合成） | 适用性 |
|------|------|------|------|
| **A 透支** | 价格离趋势多远 | ① MA200 乖离分位（= Burry 费半指标的分位化）② 距 250 日高点距离分位 ③ 60 日动量分位 | 所有板块、所有市场（骨架） |
| **B 拥挤** | 资金抱团程度 | ① 板块成交额 / 全市场成交额 → 分位（卖方拥挤度框架核心）② 相对基准 60 日相对强弱分位 | A股全部；美股 v2 仅 ② |
| **C 估值** | 贵不贵 | 板块指数 PB 或 PE 分位（**每板块指定口径**，见 §3） | 有估值历史的板块 |
| **D 专属** | 板块自己的景气/宏观锚 | 插件接口，返回一条已对齐方向的 0–100 分位序列 | v1 只定接口，v1.2 起逐板块实现 |

**方向约定**：全部对齐到"越高=越热/越危险"（顶部语义）。板块温度计 v1 只做"顶部温度"
（过热监测）；"底部温度"（板块机会）v1.1 再加——底部逻辑不是对称取反（拥挤度低≠机会，
可能是被遗忘），需要单独设计，不硬凑。

**每板块一套 = 一份配置，不是一套代码**：板块间差异全部收敛到 config 的
`weights / valuation_metric / proxy / plugins` 四个字段，计算引擎唯一。

> 指数代码与 akshare 接口覆盖**需逐一验证**（东财 `index_zh_a_hist` 对国证/中证行业码的
> 支持不齐，H 开头的中证行业码可能要走 `stock_zh_index_hist_csindex`）。验证是任务 T1。

**代理三层抽象**（数据层的关键设计，也是美股复用的关键）：

```
ProxyType = "index"        # 指数代码 → 一条 OHLCV+成交额
          | "board"        # 东财行业/概念板块名 → 一条 OHLCV+成交额
          | "basket"       # 股票代码列表 → 等权合成收盘价 + 成交额加总
```

三种类型最终都归一成同一张表：`date | close | amount`（amount 可缺）。
合成层只认这张表，不关心来源——**这就是美股能白嫖整个合成层的原因**（yfinance 出同样的表）。

## 4. 数据层设计（`quantlab/datasources/sector_source.py`，新增）

沿用 `valuation_source.py` 全部约定：akshare 函数内 import、parquet 落地 `data/sector/`、
`refresh=True` 才联网、返回 `DatetimeIndex` 序列/表。

| 函数 | 输出 | 候选 akshare 接口（**T1 验证签名**） |
|------|------|------|
| `cn_sector_ohlcv(proxy)` | `close, amount` | index → `index_zh_a_hist` / `stock_zh_index_hist_csindex`；board → `stock_board_industry_hist_em` / `stock_board_concept_hist_em`；basket → 复用现有 CN 日线源逐股取数后等权合成 |
| `cn_market_amount()` | 全市场成交额 | 中证全指(000985) 成交额做分母代理（口径与板块不完全一致没关系——取的是**比值的分位**，口径系统性偏差会被分位吸收） |
| `cn_sector_valuation(sector)` | `pb` 或 `pe` 或 `dividend_yield` | `index_value_hist_funddb(symbol, indicator)`（韭圈儿，覆盖多数行业指数）→ 备选 legulegu 系 / `stock_zh_index_value_csindex` |
| `cn_basket_close(codes)` | 等权净值 | 各股 qfq 收盘归一后等权平均；**成分固定不回填**（避免幸存者偏差幻觉，篮子从各股都有数据的最晚起点开始） |

**篮子的成交占比**：basket 类型的 amount = 成分股成交额加总，口径成立，B① 可用；
但个股停牌会造成毛刺，对 amount 做 20 日平滑后再算占比。

## 6. 输出与研判规则（两个输出，不是一个分数）

1. **温度分 0–100**：多热、多危险（弹药量）。
2. **趋势状态**：`领涨 / 走平 / 拐头`——`rs60 > +2% → 领涨；< −2% → 拐头；否则走平`
   （沿用 sector_crowding 的阈值，T6 回测再校准）。

**研判矩阵**（报告输出的人读结论）：

| | 领涨 | 走平 | 拐头 |
|---|---|---|---|
| 温度 ≥ 80 | 高危但别空（抱团在） | **减仓预警区** | **减仓触发**（高透支×拐头） |
| 60–80 | 持有观察 | 观察 | 逢反弹减 |
| < 60 | 正常 | 正常 | 正常回调 |

只有"高透支 × 拐头"给行动建议，其余全是描述——这是 T6 要回测验证的唯一规则。

## 8. A股 / 美股：公共组件与差异（组件共享答案）

| 组件 | 共享？ | 说明 |
|------|:---:|------|
| 分位/winsorize/方向对齐 | ✅ 已有 | `historical_percentile` + thermometer 模式，零改动 |
| 维度 A 透支 | ✅ 完全共享 | 只吃 close，市场无关 |
| 维度 B② 相对强弱 | ✅ 完全共享 | 换基准即可（美股用 SPY 或全市场等权） |
| 维度 B① 成交占比 | ⚠️ A股专属 | 美股拿不到干净的"全市场成交额"分母；v2 用 ETF 美元成交额自身分位替代（降级），或接 ETF 资金流（升级） |
| 维度 C 估值 | ❌ 数据层各管各 | A股走 akshare；美股指数级 PE/PB 免费源很烂，v2 先不做 |
| 维度 D 插件 | ✅ 接口共享 | 实际利率插件本来就是美股数据（TIPS），黄金板块 A/美通用 |
| 合成层 + 研判矩阵 + 报告 | ✅ 完全共享 | 只认 `date|close|amount` 表 |
| 数据层 | ❌ | `sector_source.py`(akshare) / v2 新增 `us_sector_source.py`(yfinance，SMH/GDX/SIL/XLP/COHR·LITE 篮子等) |

结论：**美股 = 新写一个数据适配器 + 板块注册表，合成层零改动**。逻辑差异只有拥挤度降级，
在"缺维度按剩余权重归一化"原则下自动处理。

## 9. 验证方案（T6，不做完不接推送）

沿用 `factor_ic_report.py` / 温度计决策层的验证习惯：

1. **单调性检验**：各板块温度分五分位桶 → 20/60 日前向收益。预期：板块层面**不单调**
   （动量强），高温桶尾部风险显著更差即可接受（同大盘结论：控回撤不增益）。
2. **规则回测**：「温度≥80 × 拐头 → 清板块仓，回落 <60 恢复」对比买入持有，
   看回撤削减和收益损耗。样本内至少覆盖 2015、2019-2021 半导体、2021 消费抱团瓦解。
3. **Burry 复现**（美股前菜，可提前）：费半 MA200 乖离全历史分位图，验证 +65% 的历史稀有度、
   以及 2000 年乖离峰值到真实顶部的时滞。产出一张图进 research/。

## 12. 已知的坑（写在前面，省得踩）

1. **东财板块历史短 + 成分漂移**：概念板块 2014± 起、成分会调（幸存者偏差内生于指数编制），
   分位窗口 <8 年的板块在报告里强制标注"分位低可信"。
2. **周期股估值方向陷阱**：半导体/有色若误配 PE 会在景气底部报过热——注册表里锁死 PB，
   代码层对 `profile=momentum/commodity` + `valuation=pe` 的组合打 warning。
3. **篮子成分固定**：不追溯调整成分。调仓 = 新起一条篮子序列，历史分位重算，git 记录变更。
4. **funddb/legulegu 源脆弱**：这类第三方聚合源接口常变，估值维度必须能整体缺席
   （缺权归一化兜底），不能让它挂掉整个温度计。
5. **银行的股息率方向**：股息率高=便宜=冷，方向与 PB 相反，合成前必须取反——测试里放一条断言。
