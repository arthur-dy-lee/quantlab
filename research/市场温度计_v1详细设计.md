# 市场温度计 v1 详细设计与任务拆解

> 范围：**仅 A股**，顶/底双温度，5 个核心指标。纯 **akshare**，离线 parquet 缓存。
> 不含美股、不含 dashboard 页（v2/v3）。v1 产出：温度计算模块 + 测试 + 一张历史温度回看图。
> 上位设计见 `市场温度计_设计.md`。

## 1. v1 指标集与权重（已按 v1 重归一化）

v1 只用 5 个指标（去掉 M2/流动性、IPO/供给），把全量权重表**按剩余项重新归一化**：

| 指标 | 维度 | 方向 | **顶部温度权重** | **底部温度权重** |
|------|------|------|:---:|:---:|
| PE 历史分位（中证全指） | 估值锚 | 高=贵 | 24% | 19% |
| ERP 股债性价比 | 估值×利率（核心） | 低=贵 / 高=便宜 | 23% | 36% |
| PB / 破净率 | 估值底 | PB高/破净少=贵 | 6% | 33% |
| 情绪/拥挤（复合） | 情绪 | 高=过热 | 47% | 12% |
| **合计** | | | **100%** | **100%** |

- **情绪复合**：`两融占流通市值分位 × 0.6 + 换手率分位 × 0.4`。
- 顶部表由情绪主导（杠杆/换手抓泡沫，破净在顶部无意义→仅 6%）。
- 底部表由 ERP+PB/破净 主导（69%，抓恐慌底）。
- 权重来源：上位设计 §4.2/§4.3 的 A股列，去掉 M2(top12/bot12)、IPO(top20/bot4)、（情绪含换手）后归一化。可在 `config.yaml` 调。

**ERP 口径（v1 简化）**：`盈利收益率 − 10年期国债` = `1/PE(沪深300) − 国债10Y(%)`。

## 3. 数据层设计（`valuation_source.py`）

每个函数：返回 `DatetimeIndex`（date）+ 数值列的 DataFrame；缓存到 `data/macro/*.parquet`；`refresh=False` 命中缓存。

| 函数 | 输出列 | 候选 akshare 接口（**需验证确切签名**） |
|------|--------|------------------------------------------|
| `cn_index_pe(index="中证全指")` | `pe` | `index_value_hist_funddb(symbol, indicator="市盈率")` 或 `stock_index_pe_lg` |
| `cn_index_pb(index="中证全指")` | `pb` | `index_value_hist_funddb(indicator="市净率")` 或 `stock_index_pb_lg` |
| `cn_below_nav()` | `below`,`total`,`ratio` | `stock_a_below_net_asset_statistics`（破净股统计） |
| `cn_margin()` | `margin_bal` | `stock_margin_sse` + `stock_margin_szse`（沪深合并） |
| `cn_float_mktcap()` | `float_mktcap` | 全市场流通市值（用于两融占比分母）；或用中证全指市值近似 |
| `cn_turnover(index="中证全指")` | `turnover` | 指数日线成交额/流通市值，或 akshare 换手率接口 |
| `cn_bond_10y()` | `y10` | `bond_china_yield`（取 10 年期）或 `bond_zh_us_rate` |
| `cn_csi300_pe()` | `pe300` | 同 PE 接口，symbol=沪深300（供 ERP） |

派生：
- `cn_erp()` → `1/pe300*100 − y10`（按日对齐，merge_asof）。
- `cn_margin_ratio()` → `margin_bal / float_mktcap`（两融占流通市值）。

**容错**：受限网络抛 `SourceUnavailable/FetchError`（沿用现有 errors）；缺某指标时上层按剩余权重归一化。

## 6. 测试策略

- **`test_thermometer.py`（不联网，合成数据）**
  - 分位单调性：递增序列 → 分位接近 100。
  - 方向对齐：同一原始序列在 top/bottom 下分值互补。
  - 缺数据归一化：去掉一个指标，权重和仍为 1。
  - winsorize 不引入未来函数（仅用过去窗口）。
  - 已知拐点回归：构造 2015 式尖顶 → 顶部温度该期显著高。
- **`test_valuation_source.py`（标 `@pytest.mark.network`）**
  - 每个取数函数返回非空、列名正确、index 单调。
  - 缓存命中：第二次调用不联网（mock 或计时）。

## 7. 验收标准（v1 Done 的定义）

1. `compute_temperature("CN","top"/"bottom")` 跑通，输出 0–100 日度序列 + 分项贡献。
2. `net_temperature` 在历史上：2007/2015 顶部温度走高、2008/2018/2024 底部温度走高（肉眼回看合理）。
3. 全部走 parquet 缓存，离线可复跑；`refresh=True` 才联网。
4. 单测通过（合成数据部分必过；network 部分可选跑）。
5. 一张历史温度叠加中证全指走势的回看图存到 `research/assets/`。

**当前读数拆解（2026-06，顶部温度 71）**

| 指标 | 现值 | 全史分位 | 贡献(顶部) |
|------|------|:---:|:---:|
| 中位 PE | 39.3 | 59 | 14.2 |
| ERP | 5.61% | 52 | 11.1 |
| 中位 PB | 2.63 | **45**（低于中位！） | 2.7 |
| 两融占比 | 1.15% | **91** | 43.1 |

> **关键结论**：当前 71 分的热度几乎全来自**两融杠杆(91 分位，贡献 43)**；估值三项（PE 59 / ERP 52 / PB 45）全在中位附近甚至偏低。
> 即「**上证 4100 是价格新高，但中位个股估值并不贵——这是杠杆推动、而非估值推动的上涨**」。净温度被便宜的估值压低，故 +27 而非 2015 式的 +95。

### 已知局限 & v1.1 改进项
1. **两融源已修**：v1 初版用 `stock_margin_sse`(仅 2018 起)致当前两融误显示为历史新高(100 分位)；已换 `macro_china_market_margin_sh`(2010 起，含 2015 峰值 0.0218≈现值 2 倍)，当前分位修正为 91。
2. **中位 vs 市值加权**：中位 PE/PB 反映「典型个股」，对上证(市值加权、银行/红利权重大)的大盘行情会低估。v1.1 可加一路指数加权 PE/PB。
3. **2016/2013 型拐点**：估值黏性 + 两融长期扩张致扩张窗口分位失真。v1.1 可评估滚动窗口分位 / 两融去趋势。
4. **破净率、换手率**：接口修复/补齐后并入情绪与估值底。

from quantlab.signals import thermometer as th

th.compute_temperature("CN", "top")          # 顶部温度 + 各指标分项贡献
# 刷新数据：任一函数加 refresh=True；出图：python scripts/market_thermometer_plot.py

### v1 两处务实简化（相对上位设计）
1. **估值底用「全市场中位数PB」替代「破净率」**——破净接口上游坏了；中位数PB 与破净率同向且更稳健。
2. **情绪 v1 只用「两融占比」(权重100%)，换手率延后 v1.1**——全市场换手率历史取数较繁，两融占比是更干净的拥挤度信号。`config.yaml` 里 `sentiment.turnover` 预留为 0。

> 影响：两融历史仅 2018 起，故 2007/2015 顶部主要由 PE/PB/ERP（2005起）支撑，2018 前的温度按「缺数据剩余权重归一化」自动处理。
